## Rule Type Reference

- **inject_prompt**: Injects text into the system prompt at the start of an iteration. Use for adding context, instructions, or constraints.
- **tool_require**: Forces a specific tool to be used when a query pattern matches (regex in pattern field).
- **format_response**: Applies formatting instructions to the response.
- **format_enforce**: Enforces strict output format validation.
- **tool_transform**: Transforms tool call arguments before the tool is invoked.
- **regex_redact**: Redacts sensitive patterns from tool results or responses (regex in pattern field, replacement text in replacement field).
- **append_text**: Appends text to results or responses.
- **block_pattern**: Blocks execution or responses containing a matched pattern (regex in pattern field).
- **require_keyword**: Requires a keyword to be present in results or responses (regex/keyword in pattern field).
- **model_switch**: Switches the LLM model for a specific iteration. Only valid at OnBeforeIteration.

## model_switch Guidance

Use model_switch to reduce cost and improve quality:
- **tool-calling / replanning iterations**: Use a lower-cost config (haiku-tier) to save budget.
- **consecutive failures ≥ 2**: Upgrade to a higher-capability config (opus-tier) for recovery.
- **final response**: Use the primary quality config (sonnet-tier) for best output.
- Set llm_config_id to a real ID from the Available LLM Configurations list.
- Only suggest model_switch when the agent has 2+ tool bindings OR archetype is data-analyst / research-assistant.
- model_switch is ONLY valid at hook_point = "OnBeforeIteration". Never suggest it at other hooks.
